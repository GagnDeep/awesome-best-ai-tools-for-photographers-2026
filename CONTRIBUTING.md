# Contributing

Thanks for contributing to `awesome-best-ai-tools-for-photographers-2026`.

## Submission Rules

- Every submission must include a public `github.com` repository URL.
- No proprietary SaaS tools or closed hosted products.
- The repository must be meaningfully useful to photographers.
- Add the following metadata from GitHub:
  - Stars
  - Primary language
  - License
  - Last visible commit or activity date
- Use the existing README format exactly.
- If a category is weak, document the gap honestly instead of padding the list.

## Suggested Submission Format

```md
#### [Tool Name](https://github.com/username/repo)
> **Description:** Short summary based on the GitHub README. Include what it does, major features, license, stars, language, and last visible activity date.

- **GitHub:** [github.com/username/repo](https://github.com/username/repo)
- **Stars:** 1234 ⭐
- **Language:** Python
- **License:** MIT
- **Last Commit:** 2026-01-01
- **Category:** tag1, tag2
- **Best for:** specific photography use case
```

## Pull Requests

- Keep descriptions factual and concise.
- Prefer README-backed claims over marketing language.
- Do not remove existing entries without explaining why.
- If metadata changed, update only the affected lines.

## Issues

Open an issue if:
- a repository was archived or deleted
- license metadata changed
- a tool is not actually open source
- a category needs to be split or marked as a gap
