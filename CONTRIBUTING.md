# Contributing

Thank you for improving this awesome list.

## Submission Rules

- Every submission must include a public `github.com` repository URL.
- No proprietary SaaS tools, cloud-only products, or closed-source services.
- The repo must be genuinely useful to photographers or adjacent imaging workflows.
- Include the current star count, primary language, license, and a short explanation of the specific photography use case.
- If a category has fewer than 5 strong repos, document that gap honestly instead of padding the list.

## Pull Request Checklist

1. Verify that the repository is public and accessible on GitHub.
2. Confirm the license from GitHub repository metadata.
3. Confirm the primary language from GitHub repository metadata.
4. Confirm recent activity from GitHub commits or activity pages.
5. Add a concise description based on the project README.
6. Keep formatting consistent with the existing README structure.

## Good Additions

- Self-hosted photo management tools
- Image restoration, upscaling, denoising, and retouching repos
- Captioning, tagging, semantic search, and curation tools
- Masking, inpainting, and object-removal tools
- Desktop or CLI utilities that wrap open source AI models for practical photographer workflows

## Not Accepted

- Tools without a GitHub repository
- Paid SaaS listings
- Repositories that are unrelated to photography workflows
- Duplicates or abandoned forks without a clear reason to include them

## Reporting Issues

Open an issue if:

- Metadata is outdated
- A license changed
- A repo became private or archived
- A better GitHub-native alternative should replace an entry
